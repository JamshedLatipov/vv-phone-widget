Can you please review the changes I have made in `OrbitalSIP/Services/SipService.cs`? I have added the following fixes:
- Added `OnCallEnded()` on `ua.ServerCallCancelled` and `ua.OnCallHungup` when receiving INVITE to fix stuck state.
- Removed unconditional BYE handling in `OnSIPRequest` which forcefully dropped any call.
- Added explicit TLS setup inside `Start()` using `SIPTLSChannel`.
- Appended `sip:` prefix in `CallAsync` and `BlindTransferAsync` when it is missing from the destination.
- Changed cleanup check during `OnRtpClosed` to also cover `CallState.OnHold`.
- Rejected pending UAS with 500 (`InternalServerError`) when `TryCreateAudio()` fails inside `AnswerAsync`.
