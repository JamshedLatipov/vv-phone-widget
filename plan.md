# Roadmap OrbitalSIP — 6-месячный план развития

Комплексный роадмап для превращения OrbitalSIP из функционального прототипа (v1.0.17) в комфортный инструмент для 8-часовой операторской смены. Разбит на 5 фаз — от ежедневных болей до стратегических фич.

---

## Фаза 1: Комфорт оператора и устранение ежедневных фрикций (месяц 1-2)

Фокус — убрать всё, что раздражает при 8-часовом использовании.

### 1.1 Переработка управления статусами
- Гранулярные паузы: 5/10/15/30/60 мин + кастомная длительность
- Кнопка "вернуться раньше" прямо на виджете
- Обратный отсчёт перерыва на тултипе виджета
- Уведомление за 1 мин до конца перерыва
- **Файлы:** `Views/StatusPopupControl.axaml(.cs)`, `Services/StatusService.cs`, `Views/WidgetView.axaml(.cs)`

### 1.2 Улучшение трансфера
- Реализовать blind transfer (SIP REFER)
- Автокомплит по списку операторов/контактов
- Список последних целей трансфера
- **Файлы:** `Views/ActiveCallView.axaml(.cs)`, `Services/SipService.cs`

### 1.3 Реализация DTMF
- Dial pad во время звонка реально отправляет DTMF тоны через SipService
- Аудио-фидбек при нажатии кнопок dial pad
- **Файлы:** `Views/ActiveCallView.axaml(.cs)`, `Services/SipService.cs`

### 1.4 Избранные контакты и быстрый набор
- Звёздочка (favorite toggle) на элементах RecentsView
- Персистенция избранных в локальный JSON
- Секция "Избранное" во вкладке Contacts (заменить плейсхолдер)
- **Файлы:** `Views/RecentsView.axaml(.cs)`, `Services/SipSettings.cs`

### 1.5 Форматирование номеров
- Авто-форматирование при вводе (+7 (999) 123-45-67)
- Валидация ввода (только цифры и +)
- **Файлы:** `Views/ExpandedView.axaml(.cs)`

### 1.6 Улучшенный фидбек копирования
- Заменить мигание иконки (1200мс) на тост-уведомление "Скопировано!" с fade-анимацией
- **Файлы:** все views с copy-кнопками, новый `Views/ToastControl.axaml(.cs)`

### 1.7 Перетаскиваемый виджет
- Позволить перетаскивание виджета на любую позицию экрана
- Запоминание позиции между перезапусками
- **Файлы:** `MainWindow.axaml.cs`, `Views/WidgetView.axaml(.cs)`

## Фаза 2: Профессиональные фичи звонков (месяц 3-4)

### 2.1 Attended Transfer
- Consultative transfer: hold текущего → звонок цели → подтверждение перевода
- SIP REFER с Replaces header
- **Файлы:** `Services/SipService.cs`, `Views/ActiveCallView.axaml(.cs)`

### 2.2 Индикатор записи
- Бейдж "REC" при серверной записи звонка (Asterisk)
- Визуальный индикатор на ActiveCallView
- **Файлы:** `Views/ActiveCallView.axaml(.cs)`, `Services/SipService.cs` (проверка SIP-заголовков)

### 2.3 Post-Call Disposition
- Модалка после завершения звонка: Решено / Эскалация / Перезвонить
- Интеграция с существующим логированием скриптов
- **Файлы:** новый `Views/PostCallView.axaml(.cs)`, `Services/CallInfoService.cs`

### 2.4 Лог пропущенных звонков
- Отслеживание missed/declined с причиной (timeout, declined, busy)
- Визуальное выделение (красная иконка) в RecentsView
- **Файлы:** `Views/RecentsView.axaml(.cs)`, `Services/SipService.cs`

### 2.5 Встроенная регулировка громкости
- Слайдер громкости в Settings: отдельно рингтон и разговор
- **Файлы:** `Views/SettingsView.axaml(.cs)`, `Services/SoundService.cs`

### 2.6 WebSocket вместо polling
- Реальное время для CDR и статистики (заменить 2-мин polling)
- Мгновенное обновление CDR при завершении звонка
- Мгновенная синхронизация статусов
- **Файлы:** новый `Services/WebSocketService.cs`, `Views/RecentsView.axaml.cs`, `Views/OperatorStatsControl.axaml.cs`
- **Зависимость:** требует поддержку WebSocket на бекенде

### 2.7 Улучшение статистики оператора
- Панель расширена по умолчанию (запоминание предпочтения)
- Метка "обновлено N мин назад"
- Мини-индикатор эффективности на bottom nav
- **Файлы:** `Views/OperatorStatsControl.axaml(.cs)`

## Фаза 3: UX-полировка и доступность (месяц 4-5)

### 3.1 Анимации и переходы
- Плавное fade open/close для попапов и модалок
- Анимация раскрытия/сворачивания панели статистики
- Scale feedback при нажатии кнопок
- **Файлы:** все .axaml файлы views

### 3.2 Настраиваемые горячие клавиши
- UI ребиндинга в Settings (частично реализовано — довести до конца)
- Детекция конфликтов с системными горячими клавишами
- **Файлы:** `Views/SettingsView.axaml(.cs)`, `Services/GlobalHotkeyService.cs`

### 3.3 Accessibility (WCAG AA)
- `AutomationProperties.Name` на все интерактивные контролы
- Фикс контраста (текст 8-9pt → минимум 11pt или увеличить контраст)
- Видимые focus-индикаторы (вернуть focus ring)
- Иконки к цветным дотам для дальтоников (не только цвет)
- **Файлы:** все .axaml файлы

### 3.4 Система тостов/уведомлений
- Переиспользуемый ToastControl для: копирование, ошибки API, смена статуса, создание лида, логирование скрипта
- Настраиваемая позиция и длительность
- **Файлы:** новый `Views/ToastControl.axaml(.cs)`, интеграция во все views

### 3.5 Улучшение модальных окон
- Focus trap (Tab цикл внутри модалки)
- Escape для закрытия
- Затемнение фона
- **Файлы:** `Views/StatusPopupControl.axaml(.cs)`, `Views/ScriptsDialog.axaml(.cs)`

### 3.6 Трекер смены и перерывов
- Отображение: время начала смены, прошло, осталось
- Подсчёт: кол-во перерывов, суммарное время перерывов
- Алерт при приближении к лимиту перерывов
- **Файлы:** новый компонент в `Views/`, `Services/StatusService.cs`

## Фаза 4: Архитектура и качество кода (месяц 5-6)

### 4.1 Dependency Injection
- Заменить статические `App.*` синглтоны на DI-контейнер (`Microsoft.Extensions.DependencyInjection`)
- Сервисы становятся тестируемыми и подменяемыми
- **Файлы:** `App.axaml.cs`, все сервисы, все views

### 4.2 Unit-тесты
- xUnit + Moq
- Покрытие: стейт-машина SIP, JWT декодирование, персистенция настроек, i18n fallback, TTL logged calls
- **Файлы:** новый тестовый проект `OrbitalSIP.Tests/`

### 4.3 CI/CD Pipeline
- GitHub Actions: build → test → publish → release
- Автоматический инкремент версии
- Замена ручного `build.ps1` на автоматизацию
- **Файлы:** новый `.github/workflows/ci.yml`

### 4.4 Безопасность
- Sentry DSN → переменная окружения (убрать хардкод из `Program.cs`)
- Фикс SSL bypass (`ServerCertificateCustomValidationCallback = (...) => true`) → валидация или конфигурируемый флаг
- Санитизация пользовательского ввода перед API-вызовами
- **Файлы:** `Program.cs`, все сервисы с HttpClient

### 4.5 Retry и обработка ошибок
- Настраиваемые HTTP таймауты
- Retry-логика через Polly для transient failures
- Видимые сообщения об ошибках пользователю (вместо тихого логирования)
- **Файлы:** все сервисы

### 4.6 Structured Logging
- Serilog вместо кастомного AppLogger
- Ротация логов по размеру и времени
- **Файлы:** `Services/AppLogger.cs`, `Program.cs`

## Фаза 5: Стратегические фичи (месяц 6+)

### 5.1 Авто-расписание онлайн/офлайн
- "Автоматически оффлайн в 17:30", "автоматически онлайн в 09:00"
- Ежедневное повторение
- **Файлы:** `Services/StatusService.cs`, `Views/SettingsView.axaml(.cs)`

### 5.2 Дашборд очереди
- Позиция в очереди, время ожидания, агенты онлайн
- Визуальный индикатор состояния очереди
- **Файлы:** новый сервис + view

### 5.3 Конференц-связь
- 3-way calling
- Объединение двух звонков в конференцию
- **Файлы:** `Services/SipService.cs` (значительная SIP-работа)

### 5.4 Глубокая CRM-интеграция
- Click-to-call: регистрация URI handler (`sip:` / `tel:`)
- Открытие страницы контакта из call info панели
- Bi-directional data sync
- **Файлы:** `Services/CallInfoService.cs`, регистрация протокола в системе

### 5.5 Панель метрик
- Тренды эффективности (день/неделя/месяц)
- Тепловая карта звонков по часам
- Трекинг персональных целей
- **Файлы:** новый `Views/DashboardView.axaml(.cs)`

### 5.6 Расширение языков
- Добавить English, Türkçe
- Подготовка к RTL (если потребуется)
- **Файлы:** новые i18n JSON, `Services/I18nService.cs`

### 5.7 Кросс-платформенность
- macOS и Linux сборки
- Абстракция platform-specific кода: `GlobalHotkeyService` (Win32 hooks), `SIPSorceryMedia.Windows`
- **Файлы:** `OrbitalSIP.csproj` (multi-target), абстрактный слой платформы

## Зависимости и допущения

| Пункт | Зависимость |
|-------|-------------|
| 2.6 WebSocket | Требуется поддержка на бекенде |
| 2.2 Индикатор записи | Asterisk должен передавать инфо о записи в SIP-заголовках |

## Открытые вопросы

1. Есть ли контроль над бекендом? Влияет на WebSocket, queue dashboard, post-call disposition
2. Нужна ли поддержка нескольких SIP-линий (multi-call) или достаточно single-call?
3. Запись звонков управляется сервером (Asterisk) или нужна клиентская запись?
4. Какие SIP-серверы используются (Asterisk, FreeSWITCH, Kamailio)? Влияет на attended transfer и conference
5. Есть ли требования по GDPR/compliance для хранения данных звонков?
