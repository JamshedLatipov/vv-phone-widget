Техническое задание: Десктопный VoIP-софтфон "Orbital SIP"
1. Общее описание
Разработка легковесного десктопного приложения для совершения аудио-звонков по протоколу SIP. Основная концепция — Widget-first UI: приложение живет в виде плавающей кнопки (FAB), которая всегда находится поверх других окон и трансформируется в полноценный интерфейс при звонке.

2. Технологический стек
Core: Tauri 2.0+ (Rust).

Frontend: React / Vue / Svelte (на выбор) + TypeScript.

VoIP Engine: SIP.js или JsSIP (через WebSocket).

Styling: Tailwind CSS + Framer Motion (для плавной анимации трансформации).

Protocol: SIP over WebSockets (WSS).

3. Функциональные требования
3.1. Режимы отображения (UI States)
Состояние "Widget" (FAB):

Круглая кнопка (~60x60px), свободно перемещаемая по экрану (Drag-and-drop).

Индикация статуса (зеленый ободок — онлайн, красный — ошибка регистрации).

Отображение аватара пользователя или иконки телефона.

Состояние "Expanded" (Dialer):

Раскрытие виджета в небольшое окно при клике.

Цифровая панель (Numpad) для набора номера.

Список последних звонков.

Состояние "Active Call":

Отображение имени/номера абонента, длительности звонка.

Кнопки: Mute (выкл. микрофон), Hold (удержание), End Call (сброс).

3.2. Телефония (VoIP)
Регистрация на SIP-сервере (Asterisk/FreePBX/Kamailio) через WSS.

Поддержка аудио-кодеков: Opus (приоритет), G.711 (alaw/ulaw).

Работа с WebRTC: корректная передача локального и удаленного медиа-потоков.

Обработка входящих вызовов в фоновом режиме (автоматическое разворачивание виджета).

3.3. Системные функции
Always on Top: Виджет всегда виден поверх других приложений (IDE, браузер).

Tray Integration: Возможность полностью свернуть приложение в системный трей.

Notifications: Нативные уведомления ОС о пропущенных звонках.

Auto-start: Опция запуска при старте системы.

4. Нефункциональные требования
Потребление ресурсов: ОЗУ не более 150-200 МБ в режиме ожидания.

Latency: Задержка звука не более 200мс (при условии стабильного интернета).

Безопасность: Хранение пароля SIP в системном хранилище (через tauri-plugin-stronghold или системный Keychain).

5. Этапы разработки (MVP)
Этап 1: Инфраструктура Tauri и Rust
Настройка прозрачного окна в tauri.conf.json.

Реализация логики Drag-region для свободного перемещения окна.

Настройка разрешений на микрофон (Permissions).

Этап 2: SIP Сигнализация
Интеграция SIP.js.

Реализация процесса регистрации (REGISTER).

Обработка статусов подключения (Connected/Disconnected).

Этап 3: Работа с Media (WebRTC)
Реализация захвата микрофона.

Привязка аудио-потока к элементу <audio> (hidden).

Управление громкостью и устройствами вывода.

Этап 4: UI/UX и Анимация
Создание CSS-анимации перехода "Круг -> Прямоугольник".

Верстка интерфейса звонка.

Реализация логики "прилипания" виджета к краям экрана.

6. Риски и решения
Проблема: Блокировка аудио браузером (Autoplay).

Решение: Использовать флаги Tauri --autoplay-policy=no-user-gesture-required.

Проблема: Потеря связи при переключении Wi-Fi/4G.

Решение: Реализовать логику Re-invite и автоматическую перерегистрацию в SIP.js.

Проблема: NAT и отсутствие звука.

Решение: Поддержка STUN/TURN серверов в конфигурации WebRTC.
